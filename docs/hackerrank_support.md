# HackerRank Support Reference

## Technical Issues
HackerRank candidates can report technical issues related to coding tests, challenge loading, and editor runtime errors.
If a candidate cannot load a challenge, they should refresh once, verify browser compatibility, and ensure stable internet connection.
For compile or runtime issues, candidates should check language version settings and test with a minimal code snippet.

## Editor and Environment
If the code editor is not responding, candidates should try a different browser or clear browser cache.
HackerRank supports Chrome, Firefox, and Edge as primary browsers for test environments.
JavaScript, Python, Java, C++, and other common languages are supported in the HackerRank editor.

## Escalation Cases
When plagiarism or cheating is suspected, the case must be escalated to trust and safety operations immediately.
Account lockouts, identity verification failures, and payment-related test purchases require human support escalation.
If a candidate is unable to submit their solution due to a platform error, escalate immediately to preserve test integrity.

## Assessment Issues
Candidates who miss a scheduled assessment due to a platform bug should contact support with evidence for rescheduling.
Test results are processed after submission and final scores are sent to recruiters automatically.
